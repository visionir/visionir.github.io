什么是MyBatis？

MyBatis 是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以使用简单的 XML 或注解来配置和映射原生类型、接口和 Java 的 POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

要使用 MyBatis， 只需将 mybatis-x.x.x.jar 文件置于 classpath 中即可。

为什么要使用MyBatis？

简化JDBC操作，提高工作效率、代码性能和程序的可维护性，降低sql与程序代码的耦合。

传统的JDBC(Java DataBase Connectivy)步骤：

  • 加载数据库驱动
  • 创建并获取数据库链接
  • 创建jdbc statement对象
  • 设置sql语句
  • 设置sql语句中的参数（使用preparedStatement）
  • 通过statement执行sql返回结果
  • 对sql执行结果进行解析
  • 释放资源，包括resultSet、preparedStatement、connection

传统的JDBC在JAVA类中添加SQL语句的硬编码方式很不方便，而Mybatis可以将SQL语句配置在XML中，通过MyBatis提供大输入参数映射方式，将参数灵活的配置在SQL语句配置文件中，同时通过MyBatis的输出映射机制，将结果集的检索自动映射成相应的JAVA对象。

综上所述，MyBatis具有采用配置文件动态管理SQL语句，输入映射、输出映射机制数据库连接池配置的持久层框架。
